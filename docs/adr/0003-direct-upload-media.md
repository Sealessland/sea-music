# ADR 0003：对象存储直传与真实媒体处理

- 状态：接受
- 日期：2026-07-13

## 决策

API 只签发短期、对象 key 隔离的 S3 上传授权，不代理视频字节。finalize 重新读取对象并验证大小、内容类型和 SHA-256。Worker 使用系统 ffprobe/ffmpeg 生成确定性 MP4 rendition 和封面，任务具备租约、续约、超时恢复及有界重试。

公开视频的播放/封面 URL 为短期签名 URL。API 对签名结果做最多 10,000 条、过期感知的进程内缓存；可用 `SEA_S3_DISABLE_DOWNLOAD_CACHE=true` 回退，便于故障隔离和 A/B 验证。

## 原因与后果

直传避免 API 带宽和内存成为瓶颈，但签名授权不能替代 finalize 校验。真实 ffmpeg 增加环境依赖，却能让验收覆盖生产路径而非模拟转码。进程内 URL 缓存不跨实例共享，且 URL 撤销依赖短 TTL。
