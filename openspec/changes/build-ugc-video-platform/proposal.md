## Why

面向 B 站及同类内容平台的后端岗位，需要一个既贴近 UGC 业务、又能展示并发控制、异步处理、数据一致性和服务治理能力的项目。相比完整复刻视频网站或堆砌微服务，本项目聚焦一条可真实运行和压测的视频投稿闭环，让关键工程决策都有可验证的业务场景。

## What Changes

- 新建一个 Go UGC 视频社区后端，覆盖账号鉴权、视频投稿、异步处理、发布和播放元数据查询。
- 提供点赞、收藏、关注、评论与弹幕等高频互动能力，并在重复请求和并发写入下保持幂等。
- 提供关注流、热门榜单和基础推荐召回，明确在线查询、缓存与离线/异步聚合的边界。
- 通过事务 Outbox 和消息消费实现可靠事件驱动流程，支持失败重试、死信处理和任务恢复。
- 提供 OpenTelemetry 指标、链路和结构化日志，以及可复现的压测与故障注入场景。
- 采用“模块化单体 API + 独立异步 Worker”起步，保留按业务边界拆分服务的接口，避免为展示而制造分布式复杂度。

## Capabilities

### New Capabilities

- `identity-access`: 用户注册登录、会话令牌、角色权限及接口限流。
- `video-publishing`: 分片/直传对象存储、投稿状态机、异步转码与审核、发布和播放元数据查询。
- `social-interactions`: 关注、点赞、收藏、评论和弹幕的写入、撤销、查询与计数聚合。
- `content-discovery`: 关注流、热门榜单和可解释的基础推荐召回与分页。
- `reliable-events`: 事务事件投递、幂等消费、重试、死信和后台任务恢复。
- `operability`: 健康检查、配置管理、结构化日志、指标、链路追踪、压测和故障演练。

### Modified Capabilities

无。

## Impact

- 新增 Go 服务端代码、数据库迁移、API 契约、异步 Worker、测试与开发环境编排。
- 运行时依赖 PostgreSQL、Redis、S3 兼容对象存储和消息代理；本地通过容器编排启动。
- 对外提供版本化 HTTP API，内部模块通过显式接口和领域事件协作。
- 项目验收将依赖真实存储、真实消息链路和可重复运行的集成/端到端测试，不以 mock 结果代替核心路径证据。
